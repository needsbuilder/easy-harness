# 보안 정책

## 취약점을 발견하셨다면

**공개 이슈로 올리지 말아주세요.** 아래로 알려주시면 됩니다.

- 메일: **hello@needslab.ai** (제목에 `[보안]`을 붙여주시면 빨리 봅니다)
- 또는 GitHub의 [Security Advisory](https://github.com/needslab-ai/easy-harness/security/advisories/new) 기능

알려주실 때 아래가 있으면 확인이 빨라집니다.

- 어떤 버전, 어떤 OS에서 생기는지
- 재현 방법
- 어떤 피해로 이어질 수 있다고 보시는지

**답장은 3일 안에 드리는 것을 목표로 합니다.** 1인이 운영하는 프로젝트라 늦어질 수 있는데,
그럴 때도 상황은 알려드리겠습니다. 확인된 취약점은 수정 후 공개하며,
원하시면 알려주신 분을 릴리스 노트에 적어드립니다.

## 지원 버전

가장 최신 릴리스만 보안 수정을 받습니다. 자동 업데이트를 켜두시길 권합니다.

## 이 앱의 보안 모델

이지 하네스는 **사용자 컴퓨터에서 설치 명령을 실제로 실행하는 앱**입니다.
그래서 아래 장치들을 두고 있습니다.

### 레시피 서명

- 무엇을 어떻게 설치할지는 `recipes/*.json`이라는 선언 파일에 적혀 있습니다.
- 원격으로 배포되는 레시피 묶음은 **ed25519로 서명**되고, 앱은 서명이 맞는 것만 읽습니다.
- 서명이 깨졌거나 없으면 원격 레시피를 버리고 **앱에 함께 들어있는 레시피로 되돌아갑니다.**
- 과거에 정당하게 서명됐던 옛 버전을 다시 밀어 넣는 **다운그레이드 공격을 막습니다.**
  캐시에 있는 것보다 버전이 높을 때만 저장합니다.
- 서명 개인키는 저장소에 없습니다. 레시피 PR이 머지되어도 서명 전에는 사용자에게 나가지 않습니다.

### 시크릿(API 키) 처리

- API 키는 메모리 안에서만 다루고 디스크에 저장하지 않습니다.
- 명령에 끼워 넣을 때만 실제 값으로 바뀌고, **화면과 로그에는 가려진 채로 나갑니다.**

### 배포 파일

- 맥 빌드는 Developer ID로 서명하고 **애플 공증(notarization)** 을 거칩니다.
- 윈도우 빌드도 서명합니다.
- 릴리스 파이프라인에 **시크릿 스캔**이 들어 있어, 걸리면 그 릴리스는 발행하지 않습니다.

## 레시피 기여를 받을 때의 원칙

레시피는 곧 실행되는 명령이라, 레시피 PR은 **보안 리뷰로 취급합니다.**
아래에 해당하면 받지 않습니다.

- 출처를 확인할 수 없는 곳에서 스크립트를 받아 실행하는 것
- 설치와 관계없는 파일을 지우거나 옮기는 것
- 사용자 몰래 자격증명이나 개인정보를 밖으로 보내는 것
- 화면에 보이는 안내 문구와 실제 실행되는 명령이 다른 것

자세한 내용은 [CONTRIBUTING.md](CONTRIBUTING.md)를 봐주세요.

## 범위 밖

- 이지 하네스가 **설치해 주는 도구들 자체의 취약점**은 각 도구 쪽에 알려주세요.
  다만 저희가 그 도구를 잘못된 방법으로 설치하고 있다면 그건 저희 문제이니 알려주시면 됩니다.
- 이미 관리자 권한을 가진 공격자가 할 수 있는 일은 범위 밖으로 봅니다.

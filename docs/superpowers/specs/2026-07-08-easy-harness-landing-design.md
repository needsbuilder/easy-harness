# 이지 하네스 소개 랜딩 사이트 + 시연영상 — 설계

- 날짜: 2026-07-08
- 상태: 설계 승인됨 (구현 계획 대기)
- 관련: 앱 리포 `easy-harness` (v0.1.2 배포 완료), 배포 리포 `needslab-ai/easy-harness-releases`

## 배경 / 목표

이지 하네스는 v0.1.2까지 실배포됐지만 제품을 소개하고 내려받게 할 **공개 웹 페이지가 없다**. 목표는 비개발자도 30초 안에 "이게 뭔지" 이해하고 자기 OS용 설치 파일을 받을 수 있는 **한 페이지 랜딩 사이트**와, 클릭 몇 번으로 AI 코딩 도구가 깔리는 과정을 보여주는 **짧은 시연영상**을 만들어 사이트에 임베드하는 것.

주 사용자는 터미널 경험이 없는 일반인이라는 전제가 모든 카피의 기준이다. 카피는 쉬운 한국어(전문용어 0), em dash 금지, 이모지는 UI 요소로 쓰지 않는다(design-system.md 규칙). 브랜드는 `design/tokens.json`의 긴고아 골드(#C9922B, 그라디언트 #E8C15A→#C9922B), 근두운 청록(#7FD4DE/#12A5B8)은 포인트로 제한, 마스코트 오공이 톤(말풍선).

## 비목표

- 블로그, 문서, 가격표, 로그인, 뉴스레터 등 (랜딩 한 페이지에 집중)
- 다국어 (한국어만)
- 실제 설치 과정을 실시간으로 캡처한 원본 화면 녹화(선택지에서 제외됨) — 하이브리드로 간다

## 결정 사항 (근거)

| 항목 | 결정 | 근거 |
|---|---|---|
| 호스팅 | Vercel + `easyharness.needslab.ai` 서브도메인 | 사용자가 이미 Vercel·needslab.ai 보유. 전문적 주소, 커스텀 도메인 용이 |
| 사이트 스택 | Vite + React + Tailwind CSS 4 | 앱과 동일 스택 → `design/tokens.json`을 Tailwind 설정으로 그대로 가져와 색·폰트가 앱과 일치. 다운로드 버튼 JS(OS 감지·API) 자연스러움 |
| 영상 방식 | 하이브리드: 실제 앱 화면 캡처 + Remotion 애니메이션 | 진짜 앱을 보여줘 정직 + 브랜드 마감. 마법사 흐름은 `demo` 모드로 깔끔 캡처 |
| 영상 소리 | 자막 중심 무음 | 랜딩 임베드 영상은 자동재생 무음이 기본 → 자막이 가장 확실히 전달 |

## 사이트 구조

리포 안에 두 디렉터리를 추가한다(앱 소스와 분리, 공유 자산은 앱의 `src/assets`·`design`에서 복사):

- `web/` — Vite + React + Tailwind 랜딩 사이트. `web/public/`에 마스코트·워드마크·favicon·`demo.mp4`·`demo-poster.png`
- `video/` — Remotion 프로젝트(컴포지션·캡처 소스). 렌더 산출물 `demo.mp4`를 `web/public/`로 복사

배포 시 Vercel 프로젝트의 루트 디렉터리를 `web/`로 지정.

### 페이지 섹션 (한 페이지, 위→아래)

1. **히어로**
   - 워드마크 + 오공이 마스코트
   - 헤드라인: "터미널 없이, 클릭 몇 번으로 AI 코딩 도구를 시작하세요"
   - 보조 카피: "복잡한 설치와 로그인은 오공이가 다 해드려요. 카드만 고르면 끝이에요."
   - 기본 CTA: 내 OS 자동 감지 다운로드 버튼(아래 §다운로드) + "다른 버전 보기"
   - 신뢰 문구: "무료로 쓸 수 있어요 · 애플 공증과 서명을 마친 안전한 앱이에요"
2. **시연영상**
   - `demo.mp4`, 자동재생·무음·반복·`playsInline`, 포스터 이미지, 창/기기 프레임 안에 배치
3. **이렇게 쉬워요 (3단계)**
   - ① 도구 고르기 — "쓰고 싶은 AI 코딩 도구를 카드에서 골라요."
   - ② 오공이가 설치·로그인 — "준비물부터 로그인까지 알아서 챙겨요."
   - ③ 바로 사용 — "설치가 잘 됐는지 확인까지 하고 끝나요."
4. **되는 도구들**
   - 하네스 배지(Claude Code·Codex·GajaeCode·OpenClaw·Hermes·OpenCode) + "플러그인과 오픈소스도 함께 챙겨요."
5. **푸터**
   - needslab.ai · 현재 버전 · GitHub 릴리스 링크 · 저작권

라이트/다크 모두 지원(토큰에 다크 색 있음), 반응형(모바일 1열). 카피는 humanize 톤 검수를 거친다.

## 다운로드 버튼 설계

문제: 릴리스 자산 이름에 버전이 박혀 있어(`Easy.Harness_0.1.2_aarch64.dmg`) 하드코딩 링크는 다음 릴리스에 깨진다.

해결: 페이지 로드시 클라이언트에서 GitHub API로 최신 릴리스를 읽어 OS/칩에 맞는 자산 URL을 동적으로 채운다.

- 엔드포인트: `https://api.github.com/repos/needslab-ai/easy-harness-releases/releases/latest` (비인증, 저트래픽이라 시간당 60회 한도 무해, 응답 캐시)
- OS 감지: `navigator.userAgentData`/`userAgent`로 mac/windows 판별
- 자산 매칭: 맥 애플실리콘 → `*_aarch64.dmg`, 맥 인텔 → `*_x64.dmg`, 윈도우 → `*_x64-setup.exe`(nsis, msi보다 친숙)
  - 브라우저에서 애플실리콘/인텔 구분이 불완전 → 기본 버튼은 애플실리콘, "인텔 맥" 링크를 "다른 버전"에 함께 노출
- "다른 버전 보기": 맥 2종·윈도우 exe/msi 전체 링크(+ 각 파일명·용량)
- 폴백: API 실패 시 버튼이 `releases/latest` 페이지로 연결 → 릴리스마다 사이트 수정 불필요
- 버전 표시: API의 `tag_name`을 히어로/푸터에 노출

## 시연영상 설계 (약 20~25초, 무음 + 자막)

### 소스 캡처
실제 앱을 `bun run tauri dev`로 띄워 핵심 화면을 캡처한다. 마법사 진행·완료 화면은 `start_flow`의 `demo` 모드(실제 실행 없이 진행 이벤트만 시뮬레이션)로 재현해 네트워크·인증 없이 깔끔하게 담는다. 캡처 대상: 카탈로그, 도구 카드, 마법사 1/4~4/4, 완료 화면. (캡처 방법은 구현 계획에서 확정 — mac `screencapture` 창 캡처 또는 컴퓨터유즈 도구)

### 장면 구성
1. (0~3s) 타이틀 카드: 워드마크 + "터미널 없이 AI 코딩 도구 시작하기" (골드 배경)
2. (3~8s) 카탈로그 화면 등장, 커서가 카드로 이동해 클릭
3. (8~16s) 마법사 진행: 진행바 1/4→4/4, 단계 자막("준비물 챙기는 중" → "설치하는 중" → "로그인 도와드릴게요" → "확인 완료"), 오공이 말풍선
4. (16~20s) 완료 화면("다 됐어요") + 오공이
5. (20~24s) 엔드카드: "지금 다운로드" + 워드마크

### 사양
- 브랜드 골드 톤, 부드러운 스프링 전환, 은은한 커서 이동, 자막은 브랜드 보이스(무음)
- 1280×720, 30fps, mp4(h264), 파일 크기 수 MB 목표(웹 임베드용)
- Remotion 버전·스캐폴딩은 구현 시 context7로 최신 확인 후 고정
- 산출물: `demo.mp4` + 첫 프레임 `demo-poster.png`

## 배포

- Vercel 프로젝트를 `easy-harness` 리포에 연결, 루트 디렉터리 `web/`, 빌드 `bun run build`(또는 Vercel 기본 Vite 감지)
- 커스텀 도메인 `easyharness.needslab.ai` 연결(needslab.ai가 Vercel이면 서브도메인 자동, 아니면 CNAME 1개). 최종 서브도메인 이름은 배포 직전 사용자 확인
- 배포는 CLI(`vercel`) 또는 깃 연결 자동 배포 중 구현 시 확정

## 확인 방법 (verification)

- 사이트: `web/`에서 빌드 통과, 브라우저에서 실제 렌더 확인(반응형·라이트/다크), 다운로드 버튼이 GitHub API로 실제 최신 자산 URL을 물어오는지 확인, 폴백 동작 확인
- 영상: Remotion 렌더가 `demo.mp4` 생성, 길이·자막 가독성·재생 확인, 페이지에서 자동재생 무음 반복 확인
- 배포: 프리뷰 URL 및 커스텀 도메인 접속 확인

## 구현 시 확정할 열린 항목

- Remotion 최신 버전·렌더 파이프라인(context7 확인 후 고정)
- 실제 앱 화면 캡처 방법(screencapture vs 컴퓨터유즈)과 캡처 해상도
- 최종 서브도메인 이름(`easyharness.needslab.ai` 기본)
- 사이트를 앱 리포 `web/`에 둘지 별도 public 리포로 뺄지(기본: `web/` 서브디렉터리)
- 배경음악 추가 여부(현재 무음, 나중에 쉽게 추가 가능)
